Qualitätssicherung

Die Qualität einer Website spielt eine wesentliche Rolle, wenn man mit ihr lernen will. Aus diesem Grund definiere ich hier eine Qualitätsmerkmale, die ich mit der Zeit auf jeder Seite anwenden will.
Eine Website muss ...

  • ... standartkonform sein (der Code muss richtig geschrieben sein)
  • ... sprachlich korrekt sein
  • ... inhaltlich korrekt sein
  • ... gut gegliedert sein
  • ... gut aussehen

Standarts

  • Alle Seiten müssen XHTML 1.0 Strict sein → W3C Validator
  • Alle Links müssen funktionieren
  • Jede Seite beginnt mit <h1> beginnen
  • <h1> gibt es nur ein mal
  • <h1> stimmt mit dem <title> überein
  • Geg.:; Ges.: werden (mit Doppelpunkt) unterstrichen
  • Die Überschriften sind hierarchisch aufgebaut (<h1>, dann <h2> ...)
  • Tabellen haben das "summary"-Attribut
  • Überschriften in Tabellen haben das "scope"-Attribut
  • Doppelpunkte werden mit unterstrichen
  • Definitionen werden im Code als solche gekennzeichnet und sind immer im Singular
